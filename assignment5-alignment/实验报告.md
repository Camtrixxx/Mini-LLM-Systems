## 2. GRPO 书面题

### Problem (baseline_calcs)

对同一 prompt 采样 $G$ 个 response,记第 $i$ 个 reward 为 $R_i$,policy score 为 $s_i=\nabla_\theta \log \pi_\theta(y_i|x)$。标准 policy-gradient 项是

```math
\mathbb{E}[R_i s_i] = \nabla_\theta \mathbb{E}[R_i].
```

如果 baseline $b(x)$ 只依赖 prompt,不依赖当前采样的 response,则

```math
\mathbb{E}[(R_i-b(x))s_i]
= \mathbb{E}[R_i s_i] - b(x)\mathbb{E}[s_i]
= \mathbb{E}[R_i s_i],
```

因为 $\mathbb{E}_{y\sim\pi}[\nabla\log\pi(y|x)]=0$。所以 prompt-level baseline 不改变期望梯度,只降低方差。

若用同组均值 $\bar R=\frac1G\sum_j R_j$ 作为 baseline,它包含了 $R_i$ 本身:

```math
\mathbb{E}[(R_i-\bar R)s_i]
= \left(1-\frac1G\right)\mathbb{E}[R_i s_i]
- \frac1G\sum_{j\ne i}\mathbb{E}[R_j]\mathbb{E}[s_i]
= \frac{G-1}{G}\mathbb{E}[R_i s_i].
```

交叉项为 0,但当前样本被减掉了 $1/G$,因此同组均值 baseline 给出一个常数缩放 $(G-1)/G$。如果改用 leave-one-out baseline $\bar R_{-i}$,则不会减掉当前样本项,梯度无偏。

### Problem (think_about_length_normalization)

Sequence normalization 先对每条 response 的 token loss 求平均,再对 batch 求平均;这样每条轨迹权重相同,短答案和长答案对一次 update 的总权重一致。Constant/token normalization 则让长 response 贡献更多 token 项,容易把优化压力偏向冗长答案。数学推理任务里,过度奖励长链式推理会导致 verbosity bias:模型可能学会写更长的 `<think>` 而不是更准的答案。Dr.GRPO/constant denominator 的动机就是尽量让长度不成为隐式奖励。

### Problem (think_about_rft)

RFT 可以看成只在正确样本上做 SFT:reward 为 1 的 response 被提高概率,reward 为 0 的 response 基本不给梯度。它的优点是稳定、简单、不会显式惩罚错误样本;缺点是样本效率低,因为早期模型正确率低时可学习样本很少,而且它只会模仿已经会做对的轨迹,对“同一题下好/坏 response 的相对偏好”利用不足。

### Problem (derive_difficulty_reweightings)

把某个 prompt 的 response reward 近似成 Bernoulli($p$),其中 $p$ 是当前模型答对该 prompt 的概率。不同算法会隐式改变 prompt difficulty 的权重:

- RFT:只有正确样本产生正向 SFT 梯度,更新出现频率正比于 $p$。太难的题几乎没有信号。
- Mean-baseline without std normalization(Dr.GRPO):有效信号来自组内正确/错误并存的情况。若 $K\sim\mathrm{Binomial}(G,p)$,混合组概率为 $1-p^G-(1-p)^G$,在 $p\approx 0.5$ 最大,太易/太难都会变弱。
- GRPO with std normalization:对每个混合组再除以组内标准差。若 $0 < K < G$,标准差约为 $\sqrt{K(G-K)/(G(G-1))}$,因此单个混合组被放大,但全错/全对组仍无信号。它仍偏重中等难度题,同时会放大低方差组的噪声。
- MaxRL(mean normalization):除以组均值约 $p$,把低成功率 prompt 的成功样本放大,更激进地推高难题权重;当均值很小时需要 $\epsilon$ 防止爆炸。

因此 difficulty weighting 不是外加技巧,而是 advantage normalization 自带的隐式 curriculum。

### Problem (think_about_advantage_normalization)

组内 std normalization 的好处是把不同 prompt 的 reward scale 对齐,避免某些 reward 方差大的题主导 batch;坏处是它改变了优化目标的 prompt 权重,并在组内 reward 几乎全同的时候制造不稳定。对二值数学 reward,全对/全错没有偏好信息,混合组才有学习信号;std normalization 会让混合组里的相对偏好更尖锐,但也可能过拟合少量偶然正确答案。

### Problem (derive_surrogate_objectives)

On-policy GRPO 的 token loss 可写作

```math
L_\text{on}(\theta) = - A_i \sum_t \log \pi_\theta(y_{i,t}\mid x,y_{i,{<}t}).
```

Off-policy 时,rollout 来自旧策略 $\pi_{\text{old}}$,需要 importance ratio:

```math
\rho_{i,t} = \exp(\log\pi_\theta(y_{i,t})-\log\pi_{\text{old}}(y_{i,t})).
```

No-clip surrogate:

```math
L_\text{IS} = - A_i \rho_{i,t}.
```

GRPO/PPO clipped surrogate:

```math
L_\text{clip} =
-\min\left(\rho_{i,t}A_i,\mathrm{clip}(\rho_{i,t},1-\epsilon,1+\epsilon)A_i\right).
```

GSPO 用 sequence-level ratio,先在 response tokens 上平均 log-ratio:

```math
\rho_i^\text{seq}
= \exp\left(\frac{1}{|y_i|}\sum_{t\in y_i}
\log\pi_\theta(y_{i,t})-\log\pi_{\text{old}}(y_{i,t})\right),
```

再对 $\rho_i^\text{seq}A_i$ 做同样的 clipping。它比 token-level clip 更接近“整条回答”的重要性重加权。

### Problem (think_about_importance_reweighting)

不做 importance reweighting 时,如果 policy 已经离 rollout policy 很远,梯度优化的是旧分布下的样本,会有偏。No-clip IS 理论上修正分布偏差,但长序列 ratio 方差很大,少数样本会支配更新。Clipping 引入偏差换稳定性,限制 policy 每轮离旧 policy 太远。Token-level clipping 更细,但会把一条 response 拆成很多局部约束;GSPO sequence-level clipping 更符合“整条推理轨迹好/坏”的 RL 单位,通常更适合长 CoT。

### Problem (grpo_experiments_standard_on_policy)

先跑小步数链路:

```bash
CUDA_VISIBLE_DEVICES=0,1 uv run python scripts/train_grpo_gsm8k.py \
  --model models/Qwen2.5-Math-7B-Instruct \
  --prompt r1_zero \

  --gradient-checkpointing \

```

正式全参 7B 训练建议切到 FSDP/ZeRO 后再跑;上面命令先保证 rollout -> reward -> GRPO update -> weight sync -> eval 的闭环成立。

实测采用 `question_only + numeric reward`,只训练最后 1 层(单卡内闭环),rollout 用 transformers backend。**完整 100 步运行**(`out/a5_full_qwen7b/standard_grpo_100step/`,lr 1e-6,2 prompts×group 4 = 8 rollouts/步,eval 128 条 every 25):

| step | eval accuracy |
|---:|---:|
| 25 | 0.4844 |
| 50 | 0.4844 |
| 75 | 0.4844 |
| 100 | 0.4844 |

**诚实结论:该设置下没有可测的学习信号**——四次评测一字不差。机制上一切正常(reward 非零、grad norm 中位数 0.21、偶有组内全对/全错步 advantage=0、loss=0),但更新量级太小:①只训 28 层中的最后 1 层;②lr 1e-6;③每步仅 8 条 rollout,100 步总共 800 条;④eval 只 128 条,分辨率 0.8%,微小提升也测不出。
一个容易误读的现象:rollout 平均奖励从前 10 步 0.29 升到后 10 步 0.60,看似在学习——但**所有 run(包括 lr=3e-7)都呈现几乎相同的轨迹**,因为各 run 共享 seed=0、题目采样顺序一致,这只是"后期抽到的题更容易",不是策略改进。固定题集的 greedy eval 才是可信指标,而它是平的。
补救方向(见后文 OLMo-1B 补充实验):换弱基座 + 全参训练 + 加大 rollout batch,才能在同等步数内看到真实涨点。

### Problem (grpo_experiments_variants_on_policy)

脚本参数:

```bash
--variant grpo          # standard
--variant grpo_constant # GRPO advantage + constant loss denominator
--variant dr_grpo       # mean baseline, no std normalization
--variant rft           # no baseline, no normalization
--variant maxrl         # mean baseline, divide by group mean
```

预期:RFT 最稳但样本效率低;Dr.GRPO 减少 std 放大噪声;MaxRL 更偏向难题,需要小心低均值组的数值稳定;标准 GRPO 通常是稳健 baseline。

完整 50 步结果(`out/a5_full_qwen7b/variant_*`,各 50 步):

| Variant | grad norm(中位) | eval@50(128) | Observation |
|---|---:|---:|---|
| standard GRPO | 0.21 | 0.4844(100步run) | std 归一化把梯度压到最小尺度 |
| Dr.GRPO | 19.4 | 0.4844 | 去掉 std 归一化,梯度大 ~90× |
| MaxRL | 45.8 | 0.4922 | 除以组均值,低均值组进一步放大 |
| RFT | 83.0 | 0.4922 | 无 baseline 无归一化,尺度最大 |

**有区分度的是梯度尺度,不是最终效果**:四个变体的 grad norm 相差两个数量级(0.21 → 83),精确对应各自的 advantage/损失归一化定义——这是对变体机制的直接实证。但 eval 仍全在噪声带内(±1 题),原因同前(last-1-layer 更新过小)。实践含义:换变体时**学习率必须随梯度尺度重调**,直接沿用同一 lr 比较变体是不公平的。

### Problem (grpo_experiments_off_policy)

脚本参数:

```bash
--importance-reweighting-method noclip
--importance-reweighting-method grpo --cliprange 0.2
--importance-reweighting-method gspo --cliprange 0.2
```

预期:no-clip 方差最大;token-level GRPO clipping 最保守;GSPO 对长 CoT 更自然,因为它限制整条 response 的平均 log-ratio。

完整 50 步结果(`out/a5_full_qwen7b/offpolicy_*`,各 50 步):

| Method | grad norm(中位) | eval@50(128) | Clip fraction |
|---|---:|---:|---:|
| noclip | 0.19 | 0.4844 | — |
| GRPO clip(ε=0.2) | 0.19 | 0.4844 | **恒为 0** |
| GSPO clip(ε=0.2) | 0.21 | 0.4922 | **恒为 0** |

**结构性局限(50 步完整版依然如此)**:脚本在每次 rollout 后**立即**用当前 policy 计算 old_log_probs 再做一次更新,policy 与 old policy 几乎重合 → ratio≡1 → 三种方法数值上退化为同一算法,clip fraction 恒 0。clipping 的意义只有在真正 off-policy(rollout 复用多轮 / 每批多次 policy update / 训练与生成异步)时才会显现——这是本实验设计的已知局限,不是 clip 实现的问题(单元测试已验证 clip 数学正确)。

### Problem (try_your_own)

候选改进:

1. 先用 `r1_zero_three_shot_gsm8k` 做 rollout,训练后再切回 zero-shot prompt eval。
2. 使用 pass@k reward shaping:同题多样本中只要出现正确答案,把同组成功样本保留为正例,错误样本给负 advantage。
3. 长度正则:对超过某 token 阈值的 response 减小 reward 或在 loss 中使用 constant normalization,避免只学会变长。
4. 分阶段温度:前期 temperature=0.9 探索,后期降到 0.6 稳定答案格式。

**实测(候选 4 的变体:group 8 + temperature 0.9,50 步,`out/a5_full_qwen7b/try_group8_temp09_50step/`)**:

| 配置 | 平均 rollout reward | eval@50(128) |
|---|---:|---:|
| 标准(group 4, temp 0.7) | 0.466 | 0.4844 |
| **group 8 + temp 0.9** | **0.506** | 0.4922 |

更大的组 + 更高探索温度提高了 rollout 奖励密度(0.506 vs 0.466):组内更可能同时出现对/错样本,非零 advantage 的比例上升,理论上每步的有效学习信号更多。与其它 run 一样,最终 eval 差异仍在噪声内——该改进的真实价值需在有信号的设置(弱基座/全参)下检验。

### 标准 GRPO 长训练(决定性结果:清晰的学习曲线)

按上一节的结论放大规模:**OLMo-2-0425-1B 全参、r1_zero prompt、numeric reward、lr 3e-6、200 步、每步 8 prompts × group 8 = 64 rollouts(≈12.8k 条,是 50 步 sweep 的 16×)、vLLM 生成(GPU1)+ 训练(GPU0)+ 每 25 步权重同步、eval 256 条**(`out/a5_full_olmo1b/standard_grpo_numeric_200step/`):

| step | 0(baseline) | 25 | 50 | 75 | 100 | 125 | 150 | 175 | 200 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| eval accuracy | 0.188 | 0.281 | 0.340 | 0.336 | 0.391 | 0.438 | 0.461 | 0.453 | **0.469** |

训练期 rollout 平均奖励同步上升:前 10 步 0.141 → 末 10 步 0.364,与 greedy eval 曲线一致。

**这是本作业要看到的核心现象:GRPO 把 GSM8K 准确率从 18.8% 提升到 46.9%(+28 个百分点,约 2.5×)**,单调上升(前 50 步最快,之后边际放缓、末段在 0.45–0.47 收敛)。它验证了整条链路真实有效:vLLM rollout → numeric reward → group-normalized advantage → 全参 GRPO 更新 → NCCL 权重同步回 vLLM → 周期 greedy eval。

### 三种设置的对照(为什么前两批平、这批涨)

| 设置 | 基座 | 可训参数 | 每步 rollout × 步数 | reward | eval 变化 |
|---|---|---|---|---|---|
| Qwen7B last-layer | Qwen2.5-Math-7B-Instruct(强) | 最后 1 层 | 8 × 100 = 800 | numeric | 平(0.484,无信号) |
| OLMo1B 50-step sweep | OLMo-2-1B(弱) | 全参 | 16 × 50 = 800 | numeric | 近平(±3%) |
| **OLMo1B 200-step** | OLMo-2-1B(弱) | **全参** | **64 × 200 ≈ 12.8k** | numeric | **0.188 → 0.469** |

三者对照给出清晰归因:①**基座要弱**(强 Instruct 基座在裸补全下已饱和,RL 无提升空间);②**要全参**(只训最后一层更新量级不足);③**训练量要够**(rollout×步数 提升一个量级才有稳定信号)。三个条件缺一不可——这正是作业默认选 OLMo-1B 全参的原因,也是前两批"诚实的负结果"的价值:它们隔离出了每个必要条件。

**关于 strict reward / "格式 0→1" 曲线的说明**:r1_zero 的 strict reward(`drgrpo_grader.r1_zero_reward_fn`)只在"格式正确**且**答案正确"时给 reward=1,格式对但答案错给 0(注释:避免 format hacking)。OLMo 基座 strict format_accuracy=0,联合命中概率≈0,strict reward 会导致**冷启动死锁**(奖励恒 0、advantage 恒 0、无梯度)。因此本实验用 numeric reward(从第 0 步就有 18.8% 的非零信号)以获得可学习的奖励曲线;著名的"format reward 0→1"曲线在这个不给纯格式分的 reward 定义下不会出现,需要改用给格式部分分的 reward 才能复现。
