# Stanford CS336: Language Modeling from Scratch — 学习指南

> Spring 2026 版 | 整理日期：2026-07-05

## 课程概况

- **官网**：https://cs336.stanford.edu/ （含完整 schedule 和讲义）
- **讲师**：Percy Liang & Tatsunori Hashimoto
- **视频**：[Spring 2026 YouTube 播放列表](https://www.youtube.com/playlist?list=PLoROMvodv4rMqXOcazWaTUHhq-yembLCV)
- **讲义代码**：https://github.com/stanford-cs336/lectures （可执行的 lecture traces）
- **强度**：5 学分，约 20–25 小时/周。官方原话：Assignment 1 的工作量 ≈ CS224n 全部作业 + 期末项目
- **理念**：像"从零写操作系统"一样，从零构建一个完整的 LLM 技术栈——tokenizer、模型、训练系统、数据管线、对齐，全部亲手实现

## 五个作业一览（均为 2026 版）

| # | 目录 | 主题 | 核心任务 |
|---|------|------|---------|
| 1 | `assignment1-basics` | 基础 | 从零实现 BPE tokenizer、Transformer LM（RMSNorm/RoPE/SwiGLU）、AdamW、训练循环；在 TinyStories / OpenWebText 上训练 |
| 2 | `assignment2-systems` | 系统 | 性能剖析与基准测试、用 Triton 手写 FlashAttention-2、手写 DDP 数据并行与优化器状态分片 |
| 3 | `assignment3-scaling` | 缩放定律 | 拟合 scaling laws、在给定算力预算下预测最优超参。⚠️ 部分任务依赖斯坦福内部训练 API，自学时可只做本地部分，或读 PDF + 用公开数据复现拟合 |
| 4 | `assignment4-data` | 数据 | 从 Common Crawl WET 文件构建预训练数据管线：HTML 抽取、语言识别、质量过滤、去重（MinHash） |
| 5 | `assignment5-alignment` | 对齐 | 数学推理 RL（含 GRPO）、SFT、安全 RLHF（附补充 PDF） |

**建议顺序**：1 → 2 → 4 → 5，(3 可穿插)。A2 依赖 A1 的模型代码（仓库里自带参考实现 `cs336-basics`），A4/A5 相对独立。

每个作业目录里都有：
- `cs336_assignment*.pdf` —— 作业讲义（先读这个！）
- `tests/` —— 单元测试；`tests/adapters.py` 是你的实现与测试的对接层
- `cs336_*/` —— 你写代码的包目录
- `CHANGELOG.md` —— 作业勘误更新

**工作流**：读 PDF → 在 `cs336_*/` 包里实现 → 在 `tests/adapters.py` 中把测试接到你的实现 → `pytest` 验证（初始状态所有测试都是 `NotImplementedError`）。

## 环境

- **conda 环境**：`hyh-cs336`（Python 3.12.13）—— 2026 版作业要求 Python ≥3.12（A5 要求 <3.13）
- ⚠️ **torch 必须装 cu128 版**：本机驱动是 CUDA 12.8，PyPI 默认的 torch（cu130）无法使用 GPU。已装 `torch 2.11.0+cu128`。以后其他作业装依赖时若 torch 被覆盖，用：
  ```bash
  pip install torch==2.11.0 --index-url https://download.pytorch.org/whl/cu128 --force-reinstall
  ```
  ```bash
  conda activate hyh-cs336
  cd assignment1-basics
  pytest   # 跑测试
  ```
- 每个作业按需 `pip install -e .`（A1 已装）。若日后遇到跨作业依赖冲突，可改用课程官方的 `uv`（每仓库独立锁定环境）：`pip install uv && uv run pytest`
- **本机硬件**：4× A800-80GB —— 训练（A1/A4/A5）、多卡 DDP（A2）都完全够用；调试先用 CPU/小模型，训练再上 GPU
